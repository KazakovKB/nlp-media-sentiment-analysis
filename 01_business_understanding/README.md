# Бизнес-анализ проекта  
## NLP-система анализа общественных настроений в медиапространстве

---

## Организационная структура

**Название проекта:**  
Система автоматического анализа общественных настроений на основе новостных публикаций и социальных сообщений.

**Цель проекта:**
Создать инструмент, который позволяет в реальном времени отслеживать и анализировать общественные настроения
в медиапространстве, выявлять всплески негатива и позитивные тренды, поддерживая процесс принятия решений PR и аналитических служб.
Основная идея – автоматизация ручного анализа медиа-контента, чтобы сократить затраты времени аналитиков и повысить точность оценки общественных настроений.

**Основные пользователи:**
- PR-специалисты – для оперативного реагирования на информационные всплески.  
- Аналитики – для анализа динамики общественных настроений и формирования отчётов.  

**Команда проекта:**
- **Data Scientist (исполнитель)** — отвечает за полный цикл работ, например: анализ, сбор и хранение данных, разработку, развёртывание и оптимизацию ML-моделей.

**Существующие решения:**
В настоящее время мониторинг общественных настроений проводится вручную или с использованием инструментов, не имеющих встроенного анализа тональности. Такие решения не обеспечивают:
- автоматическую классификацию тональности текстов;  
- выявление эмоциональных трендов;
- визуализацию динамики настроений по времени.

## Решаемые задачи с точки зрения аналитики (Data Mining goals)

**ML-задача:**  
Классификация текстов по тональности (позитив / нейтрально / негатив).

**Тип задачи:**  
Supervised Learning – многоклассовая классификация.

**Метрики оценки:**
- `Accuracy`  
  - Даёт общую картину качества модели;
  - Хорошо читается для бизнеса;
  - Есть ограничения в случае если классы несбалансированны.
- `Precision`, `Recall`, `F1-score`
  - Детальная оценка по каждому классу и общий баланс.

**Критерии успешности:**
- Минимальный порог: `F1 > 0.74`  
- Оптимальный результат: `F1 > 0.79`

**Baseline:**  
Logistic Regression + TF-IDF векторизация.

**Улучшение:**  
Использование предобученной трансформер-модели RuBERT.

## План проекта (Project Plan)

Проект реализуется по методологии **CRISP-DM** и включает шесть фаз.  
Общая продолжительность — **8 недель**.

| Фаза | Цель | Основные действия                                        | Сроки |
|------|------|----------------------------------------------------------|-------|
| **1. Business Understanding** | Определение целей, KPI и рисков | Анализ текущей ситуации, подготовка бизнес-анализа       | Неделя 1 |
| **2. Data Understanding** | Сбор и исследование данных | Получение и описание данных, первичный EDA               | Неделя 2 |
| **3. Data Preparation** | Подготовка данных для моделирования | Очистка, токенизация, лемматизация, балансировка классов | Неделя 3–4 |
| **4. Modeling** | Обучение и подбор модели | TF-IDF + Logistic Regression, затем BERT                 | Неделя 5–6 |
| **5. Evaluation** | Оценка результатов | Анализ метрик, сравнение моделей, интерпретация          | Неделя 7 |
| **6. Deployment** | Внедрение и демонстрация | Реализация простого веб-сервиса                          | Неделя 8 |
